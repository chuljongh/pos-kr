# 브로셔 제작·재생성 노트 (`brochure1.html`)

pos.kr 모바일 QR 식수관리시스템 **A4 2페이지 인쇄용 브로셔**. `index.html` 내용을 압축하고 PAYCO 레드 디자인 시스템을 적용한 자체 완결형(임베디드 CSS) HTML 1파일.

위치: 프로젝트 루트 `brochure1.html` (`brochure2.html`은 히어로 헤드라인·04 섹션 문구가 다른 변형)

---

## 1. 구성 (2페이지)

- **페이지 1 (후킹·도입 이유)**: 히어로 배너(현장 사진) → 레드 stat 카드 `0원/0%/1분` → ① 왜 모바일 식수관리? (경제성·편의성·신뢰성) → ② 기존 RFID·바코드와 비교표
- **페이지 2 (방법·행동)**: ③ 도입(현장별 결제) → ④ 결제 상황 지원(식대 정책·관리 대행) → ⑤ 3단계 시작 → ⑥ FAQ → 레드 CTA + 푸터(사업자 정보)

섹션 제목은 사장님의 속마음 **질문이 이어지는 흐름**으로 구성(단절감 방지).

## 2. 사용 자산 (프로젝트 루트)

| 파일 | 용도 |
|---|---|
| `두리미가_QR스캔.jpg` | 히어로 배너 배경 (고객 QR 폰 + 매장 스캐너 합성본) |
| `손스마트폰_QR합성.png` | 손+폰(QR 화면) 투명 컷아웃 (예비 자산) |
| 원본: `두리미가.jpg`, `QR코드_홍길동.jpeg`, `손스마트폰.png`, `두리미가_스캐너내부.jpg` | 합성 재료 (보존) |

디자인 토큰·가이드 출처: `payco-design-system/` (브랜드 레드 `#FF2233`).

## 3. 합성 이미지 재생성 방법 (비자명 — 스크립트는 일회용으로 삭제됨)

`QR코드_홍길동.jpeg`를 `손스마트폰.png` 화면에 넣고, 매장(`두리미가.jpg`) 좌측 하단에 배치하는 방식. **도구: Python + Pillow(PIL)만** 사용 (cv2/numpy/ImageMagick 없음).

1. **폰 화면 4모서리 좌표** (flood-fill로 검출, `손스마트폰.png` 3000×1976 기준):
   - TL `(1123, 229)` · TR `(1787, 235)` · BR `(1783, 1650)` · BL `(1118, 1645)`
2. **QR 왜곡 방지**: QR을 폰 화면 비율로 늘리지 말 것. 흰 화면 사각형(쿼드 비율 ≈ 1:2.13)을 만들고 그 위에 QR을 **비율 유지로 중앙 배치**한 뒤 통째로 원근 변환.
3. **원근 변환**: PIL `Image.transform(size, Image.PERSPECTIVE, coeffs, BICUBIC)`. 계수는 dest(폰 쿼드)→src(화면 사각형) 8점쌍으로 **8×8 선형계를 직접 가우스 소거**해 계산(numpy 없이). 쿼드 폴리곤 마스크(중심으로 3px 인셋)로 화면 영역만 합성.
4. **매장 합성**: 합성한 손-폰을 alpha bbox로 크롭 → 높이 ≈ 매장 높이의 0.66 → **좌측 하단**(x ≈ -5%BW, y = 바닥 정렬) 배치 + 소프트 그림자(GaussianBlur).

> 좌표/배치 비율만 바꾸면 새 QR·다른 위치로 재생성 가능. 검증은 작은 미리보기 JPG로 눈으로 확인.

## 4. 인쇄 / PDF 변환

브라우저에서 `brochure1.html` 열고 **Ctrl+P** → 용지 **A4**, 여백 **"없음"** → 2페이지 PDF.
`@page { size:A4; margin:0 }` + `print-color-adjust:exact`로 레드/사진 배경이 그대로 인쇄됨.

## 6. 레이아웃 구조 메모

- `.page`는 **고정 높이 297mm + flex 컬럼**. 섹션 시작 요소(`.sec-head`, `.faq`, `.cta`)에 `margin-top:auto`를 줘서 **남는 세로 공간을 섹션 사이로 균등 분배** → 콘텐츠를 가감해도 자동 재배치되고 하단 빈 공간이 안 생김.
- 히어로 배너는 풀블리드(`margin: -14mm -15mm ...`), 상단 어두운 그라데이션 위 흰 헤드라인. 사진 위 `strong`은 검정으로 사라지므로 `.hero-banner strong{color:#fff}`로 오버라이드.
- 비교표: ours = 레드 ✓ + 옅은 핑크 배경, theirs = 회색 ✗ + 취소선. 값 색은 대비 위해 `--red-darker` 사용.
